# 權益授權證明

### DPoS 就像是代議民主。你手上有選票（代幣），但不需要親自去國會（驗證區塊），只要投票選出你信任的代表就好。這些被選出的代表負責維護網路，作為回報會分享獎勵給你。

---

## 什麼是權益授權證明？

| 概念 | 說明 |
|------|------|
| 英文 | Delegated Proof of Stake（DPoS）|
| 發明者 | Daniel Larimer（2014）|
| 代表項目 | EOS、TRON、BitShares |
| 特色 | 極快、但較中心化 |

---

## 運作原理

### 基本機制

```
DPoS 的運作方式：

1. 代幣持有者投票
   ↓
2. 選出固定數量的「代表」
   （EOS 是 21 個，TRON 是 27 個）
   ↓
3. 代表輪流打包區塊
   ↓
4. 代表獲得區塊獎勵
   ↓
5. 代表分享獎勵給投票者
   ↓
6. 表現差的代表會被投票淘汰
```

### 關鍵角色

| 角色 | 說明 |
|------|------|
| 代幣持有者 | 用持幣投票選代表 |
| 代表/見證人 | 被選中驗證交易的節點 |
| 備選節點 | 隨時準備接替的候補 |

{% hint style="info" %}
**投票權重**

持有越多代幣，投票權重越大。這和股份公司的投票權類似——大股東說了算。
{% endhint %}

## 優缺點

### 優點

| 優點 | 說明 |
|------|------|
| 極快 | EOS 可達數千 TPS |
| 節能 | 只有少數節點驗證 |
| 治理機制 | 社群可以投票換掉壞代表 |
| 獎勵分享 | 代表會分享收益給投票者 |

### 缺點

| 缺點 | 說明 |
|------|------|
| 中心化 | 只有少數節點控制網路 |
| 財閥政治 | 大戶可能串通控制 |
| 投票冷漠 | 很多人不投票 |
| 賄選風險 | 代表可能用錢買票 |

---

## DPoS vs PoS

| 比較 | DPoS | PoS |
|------|------|-----|
| 驗證者數量 | 固定少數（21-101）| 無上限 |
| 選擇方式 | 投票選舉 | 隨機選擇 |
| 速度 | 更快 | 較慢 |
| 去中心化 | 較低 | 較高 |
| 治理 | 明確的投票機制 | 依項目而異 |

---

## 代表項目

### EOS

| 資訊 | 說明 |
|------|------|
| 代表數量 | 21 個 BP（Block Producer）|
| 選舉方式 | 持續投票 |
| 爭議 | 中國節點曾主導、賄選問題 |

### TRON

| 資訊 | 說明 |
|------|------|
| 代表數量 | 27 個超級代表 |
| 投票獎勵 | 投票者可獲得分潤 |
| 爭議 | 中心化程度高 |

```
DPoS 的中心化爭議：

EOS 的情況（2018-2019）：
- 21 個 BP 中，多個疑似同一團隊控制
- 中國節點曾佔多數
- 被批評「不夠去中心化」

反駁觀點：
- 至少有投票機制
- 表現差可以被換掉
- 比完全中心化好
```

{% hint style="warning" %}
**DPoS 的風險**

因為只有少數節點：
- 容易被政府施壓
- 可能串通審查交易
- 網路安全依賴少數人

適合重視速度勝過去中心化的應用。
{% endhint %}

## 投票參與

### 為什麼要投票？

| 理由 | 說明 |
|------|------|
| 獲得獎勵 | 很多代表會分潤給投票者 |
| 影響治理 | 選出好的代表維護網路 |
| 責任 | 持幣就有責任參與治理 |

### 投票策略

| 策略 | 說明 |
|------|------|
| 研究代表 | 了解他們的技術能力和信譽 |
| 分散投票 | 不要把票都投給一個代表 |
| 持續關注 | 如果代表表現差，換票 |
