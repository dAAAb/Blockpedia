# 工作量證明

### 工作量證明就像一場解數學題比賽。誰先解出來，誰就有權記帳，並獲得獎勵。這道題很難解，但答案很容易驗證。比特幣的「挖礦」，就是礦工們在比賽解題。

---

## 什麼是工作量證明？

| 概念 | 說明 |
|------|------|
| 英文 | Proof of Work（PoW）|
| 發明者 | 最早概念由 Adam Back 提出（1997）|
| 應用 | Bitcoin、Litecoin、Dogecoin |
| 俗稱 | 挖礦 |

---

## 運作原理

### 簡化版流程

```
工作量證明的流程：

1. 礦工收集待處理交易
2. 開始「猜」一個數字（nonce）
3. 將交易+nonce 做雜湊運算
4. 如果雜湊值符合難度要求 → 獲勝
5. 其他礦工驗證答案
6. 區塊加入鏈上，礦工獲得獎勵
7. 重新開始
```

### 難度調整

| 機制 | 說明 |
|------|------|
| 目的 | 維持穩定的出塊時間 |
| Bitcoin | 每 2016 區塊調整一次 |
| 目標 | 約 10 分鐘一個區塊 |

{% hint style="info" %}
**為什麼需要難度調整？**

如果礦工越來越多，算力變強，沒有調整的話區塊會生成太快。難度調整確保無論多少人挖礦，出塊速度維持穩定。
{% endhint %}

## 優缺點

### 優點

| 優點 | 說明 |
|------|------|
| 安全性高 | 攻擊成本極高（需要 51% 算力）|
| 去中心化 | 任何人都可以參與挖礦 |
| 經過驗證 | 比特幣運作超過 15 年 |

### 缺點

| 缺點 | 說明 |
|------|------|
| 耗電 | Bitcoin 年耗電量約等於一個小國家 |
| 速度慢 | Bitcoin 每秒約 7 筆交易 |
| 專業化 | 現在需要專業礦機才能競爭 |

## 挖礦的演變

```
挖礦設備演變：

2009：CPU 挖礦（一般電腦）
      ↓
2010：GPU 挖礦（顯示卡）
      ↓
2013：ASIC 礦機（專用晶片）
      ↓
現在：礦場、礦池主導
```

### 礦池

| 概念 | 說明 |
|------|------|
| 定義 | 多個礦工聯合起來一起挖 |
| 好處 | 收益更穩定（按算力分配）|
| 大礦池 | Foundry、Antpool、F2Pool |

## 51% 攻擊

### 什麼是 51% 攻擊？

```
51% 攻擊：

如果有人控制超過 50% 的算力：
- 可以雙花（同一筆錢花兩次）
- 可以阻止其他交易確認
- 但不能偷別人的幣

比特幣為什麼安全？
- 需要天文數字的算力
- 攻擊成本遠超收益
- 經濟上不理性
```

{% hint style="warning" %}
**小型 PoW 鏈的風險**

Bitcoin 很安全，但小型 PoW 區塊鏈容易被攻擊。歷史上 Bitcoin Gold、Ethereum Classic 都遭受過 51% 攻擊。
{% endhint %}

## PoW vs PoS

| 比較 | PoW | PoS |
|------|-----|-----|
| 資源消耗 | 電力 | 質押代幣 |
| 環保 | 爭議大 | 較環保 |
| 進入門檻 | 需要礦機 | 需要代幣 |
| 代表 | Bitcoin | Ethereum |
