# 授權證明

### 授權證明就像是一個「VIP 俱樂部」。只有被認可的成員（授權節點）才能參與驗證。這種方式犧牲了去中心化，換取速度和效率。適合企業內部使用，或是測試網路。

---

## 什麼是授權證明？

| 概念 | 說明 |
|------|------|
| 英文 | Proof of Authority（PoA）|
| 核心概念 | 只有授權節點可以驗證 |
| 應用場景 | 私有鏈、聯盟鏈、測試網 |
| 特色 | 極快、但高度中心化 |

---

## 運作原理

### 基本機制

```
PoA 的運作方式：

1. 預先指定一組「授權驗證者」
   （通常是已知身份的機構）
   ↓
2. 只有這些驗證者可以打包區塊
   ↓
3. 驗證者輪流出塊
   ↓
4. 其他節點只能接收和驗證

關鍵：驗證者身份是公開的
     → 用「聲譽」作為擔保
```

### 授權驗證者

| 特點 | 說明 |
|------|------|
| 身份已知 | 不是匿名的 |
| 數量有限 | 通常少於 20 個 |
| 信任基礎 | 用聲譽和身份擔保 |

{% hint style="info" %}
**PoA 的信任模型**

在 PoW/PoS 中，我們信任「機制」（數學和經濟激勵）。在 PoA 中，我們信任「人」（已知身份的驗證者）。這是根本性的差異。
{% endhint %}

## 優缺點

### 優點

| 優點 | 說明 |
|------|------|
| 極快 | 不需要複雜的共識過程 |
| 節能 | 不需要挖礦或質押 |
| 穩定 | 驗證者固定，不會波動 |
| 適合企業 | 符合合規需求 |

### 缺點

| 缺點 | 說明 |
|------|------|
| 高度中心化 | 少數人控制網路 |
| 需要信任 | 必須信任驗證者 |
| 不抗審查 | 驗證者可以拒絕交易 |
| 非「真正的」區塊鏈 | 某些人認為這不算區塊鏈 |

---

## 使用場景

### 適合 PoA 的情況

| 場景 | 說明 |
|------|------|
| 企業內部鏈 | 公司內部的資料共享 |
| 聯盟鏈 | 多家企業合作的區塊鏈 |
| 測試網 | Ethereum 的 Goerli 測試網 |
| 供應鏈 | 追蹤商品流向 |

### 不適合 PoA 的情況

| 場景 | 原因 |
|------|------|
| 公開貨幣 | 需要去中心化 |
| 抗審查應用 | PoA 無法抗審查 |
| 匿名需求 | 驗證者身份是公開的 |

```
PoA 的典型應用：

企業區塊鏈：
├── 金融機構結算
├── 供應鏈追蹤
├── 醫療記錄共享
└── 政府資料存證

這些場景的共同點：
- 參與者彼此認識
- 需要高速度
- 不需要完全去中心化
- 有合規要求
```

## PoA 網路範例

### Ethereum 測試網

| 測試網 | 說明 |
|--------|------|
| Goerli | 使用 PoA 的測試網 |
| Sepolia | 目前主要的測試網 |

### 企業區塊鏈

| 平台 | 說明 |
|------|------|
| Hyperledger | 企業區塊鏈框架 |
| VeChain | 供應鏈區塊鏈（使用 PoA 變體）|

{% hint style="warning" %}
**PoA 的局限**

PoA 本質上是「中心化」的，驗證者可以：
- 串通作惡
- 審查特定交易
- 被政府強制關閉

如果你的應用需要抗審查，PoA 不是正確的選擇。
{% endhint %}

---

## PoA vs 其他共識

| 比較 | PoA | PoW | PoS |
|------|-----|-----|-----|
| 去中心化 | 極低 | 高 | 中 |
| 速度 | 極快 | 慢 | 快 |
| 能耗 | 低 | 高 | 低 |
| 信任基礎 | 身份/聲譽 | 算力 | 質押 |
| 適用場景 | 企業/私有 | 公開貨幣 | 公開鏈 |
