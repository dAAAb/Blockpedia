# 權益證明

### 如果工作量證明是「誰算力強誰說了算」，權益證明就是「誰持幣多誰說了算」。你不需要買礦機，只需要把代幣「質押」在網路上，就有機會獲得記帳權和獎勵。2022 年以太坊從 PoW 轉向 PoS，是區塊鏈歷史上的大事件。

---

## 什麼是權益證明？

| 概念 | 說明 |
|------|------|
| 英文 | Proof of Stake（PoS）|
| 核心概念 | 質押代幣獲得驗證權 |
| 代表項目 | Ethereum、Cardano、Solana |
| 特色 | 節能、較快 |

---

## 運作原理

### 基本流程

```
權益證明的流程：

1. 驗證者質押代幣（如 ETH）
2. 系統隨機選擇驗證者
3. 被選中者打包區塊
4. 其他驗證者確認
5. 驗證者獲得獎勵
6. 如果作惡 → 質押被罰沒

關鍵：質押越多，被選中機率越高
```

### 質押（Staking）

| 概念 | 說明 |
|------|------|
| 質押 | 把代幣鎖定在智能合約中 |
| 目的 | 作為「保證金」防止作惡 |
| Ethereum | 需要 32 ETH 成為驗證者 |

{% hint style="info" %}
**質押門檻太高怎麼辦？**

32 ETH（約數萬美元）對一般人太高。解決方案：
- 質押服務（如 Lido、Coinbase）
- 質押池（多人合資）
- 流動性質押（stETH）
{% endhint %}

## 優缺點

### 優點

| 優點 | 說明 |
|------|------|
| 節能 | 能耗約 PoW 的 0.01% |
| 更快 | 交易確認速度更快 |
| 進入容易 | 不需要專業設備 |
| 經濟激勵 | 持幣者獲得被動收入 |

### 缺點

| 缺點 | 說明 |
|------|------|
| 富者越富 | 持幣多的人獲得更多獎勵 |
| 攻擊風險 | 理論上可以買下足夠代幣攻擊 |
| 中心化風險 | 大質押池可能過於集中 |
| 複雜性 | 機制設計比 PoW 複雜 |

## Ethereum 的 PoS 轉型

### The Merge（合併）

```
Ethereum 的轉型歷程：

2020/12：Beacon Chain 啟動（PoS 測試）
2022/09：The Merge 完成
         主網正式轉向 PoS

影響：
- 能耗降低 99.95%
- 不再有 ETH 挖礦
- 礦工轉向其他鏈（如 ETC）
```

### 驗證者統計

| 數據 | 說明 |
|------|------|
| 驗證者數量 | 超過 90 萬 |
| 質押總量 | 超過 3000 萬 ETH |
| 年化收益 | 約 3-5% |

## Nothing at Stake 問題

### 什麼是 Nothing at Stake？

```
PoS 的理論問題：

在 PoW 中：
- 分叉時，礦工只能選一邊挖
- 因為算力是有限資源

在 PoS 中：
- 分叉時，驗證者可以兩邊都押注
- 反正「沒有成本」

解決方案：
- Slashing（罰沒機制）
- 如果發現兩邊押注，沒收質押
```

{% hint style="warning" %}
**驗證者風險**

成為驗證者要注意：
- 離線太久會被罰
- 惡意行為會被沒收質押
- 需要穩定的設備和網路
{% endhint %}
