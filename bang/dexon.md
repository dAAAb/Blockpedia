# DEXON

### DEXON 曾是台灣區塊鏈產業的驕傲。技術團隊強、募資成功、還有交易所支持。但最終它隨著 COBINHOOD 的崩潰而消失。這是一個關於「技術不是一切」的故事。

## DEXON 的技術

### 創新之處

| 技術 | 說明 |
|------|------|
| 共識 | DAG-based Byzantine Agreement |
| 速度 | 號稱可達 10,000+ TPS |
| 最終性 | 快速確認 |
| 架構 | 多鏈結構 |

### 技術特點

```
DEXON 的技術亮點：

共識機制：
- 使用 DAG（有向無環圖）
- 拜占庭容錯
- 隨機信標

設計目標：
- 高吞吐量
- 低延遲
- 公平排序

團隊背景：
- 台大資工背景
- 發表過學術論文
- 技術實力受認可
```

{% hint style="info" %}
**技術實力**

DEXON 團隊的技術論文曾發表在學術會議上。這是台灣少數有原創技術的區塊鏈項目。
{% endhint %}

## 興衰歷程

### 時間線

| 時期 | 事件 |
|------|------|
| 2017 | COBINHOOD 交易所成立 |
| 2018 | DEXON 開始開發 |
| 2019 年初 | DEXON 主網上線 |
| 2019 年中 | COBINHOOD 財務問題浮現 |
| 2019 年底 | COBINHOOD 宣布關閉 |
| 2020 | DEXON 實質停止 |

### 崩潰原因

```
DEXON/COBINHOOD 崩潰原因：

1. 母公司財務問題
   - COBINHOOD 燒錢過快
   - 2018 熊市收入驟降
   - 資金鏈斷裂

2. 商業模式問題
   - 交易所競爭激烈
   - 公鏈開發成本高
   - 兩線作戰資源不足

3. 市場時機
   - 2018-2019 熊市
   - 投資人撤退
   - 用戶流失

4. 治理問題
   - 內部管理爭議
   - 團隊士氣受影響
```

{% hint style="warning" %}
**關聯風險**

DEXON 的失敗主要不是技術問題，而是母公司 COBINHOOD 的財務崩潰。這提醒我們：項目背後的公司和資金狀況同樣重要。
{% endhint %}

## 教訓

### 對投資者

| 教訓 | 說明 |
|------|------|
| 技術不等於成功 | 好技術可能敵不過壞管理 |
| 關注母公司 | 子項目依賴母公司資源 |
| 熊市風險 | 小團隊難以度過長熊市 |
| 分散投資 | 不要重倉單一項目 |

### 對台灣區塊鏈

| 教訓 | 說明 |
|------|------|
| 商業模式重要 | 技術要配合可持續商業模式 |
| 資金規劃 | 要有足夠的 runway |
| 專注 | 不要同時做太多事 |
| 社群 | 建立強大的開發者社群 |
