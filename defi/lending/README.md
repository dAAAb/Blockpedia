# DeFi 借貸
> **最後更新：2026 年 1 月**

> ⚠️ **免責聲明**：本文僅供教育目的，不構成投資建議。加密貨幣投資有風險，請自行研究（DYOR）後再做決定。

## DeFi 借貸讓你不需要銀行就能存款賺利息、抵押借款，一切由智能合約自動運作。

想借錢？

去銀行：填表格、交資料、等審批、簽合約。
可能被拒絕。可能要等一週。

去 DeFi：連接錢包、存入抵押品、借出資金。
幾分鐘搞定。沒有人可以拒絕你。

這就是 **DeFi 借貸**。

### 什麼是 DeFi 借貸？

DeFi 借貸是運行在區塊鏈上的借貸市場。

**存款（Lending）**
- 把你的加密貨幣存入協議
- 自動賺取利息
- 隨時可以提出

**借款（Borrowing）**
- 存入抵押品
- 借出其他資產
- 支付利息

### 運作機制

**流動性池模型**

大多數 DeFi 借貸協議使用「流動性池」：

1. **存款者**把資產存入池中
2. **借款者**從池中借出資產
3. **利率**根據供需自動調整
4. **利息**從借款者流向存款者

沒有對手方匹配，只有算法和智能合約。

### 超額抵押

DeFi 借貸的核心是**超額抵押**。

**為什麼需要超額抵押？**

- DeFi 是匿名的，不知道你是誰
- 沒有信用評分
- 沒有法律追討機制
- 需要其他方式確保還款

**怎麼運作？**

- 借 $100，需要存入 $150 的抵押品
- 如果你不還款，協議可以清算你的抵押品
- 超額的部分提供安全緩衝

**抵押率（LTV）**

Loan-to-Value，貸款與價值的比率：
- 抵押品值 $150，借 $100
- LTV = 100/150 = 66.7%

不同資產有不同的 LTV 上限。

### 清算

當抵押品價值下跌到危險水平時，會發生**清算**。

**例子**

- 你抵押 1 ETH（$2,000），借 $1,000
- ETH 跌到 $1,100
- 你的抵押率太低，觸發清算
- 部分（或全部）抵押品被賣出還債

**清算懲罰**

被清算時通常要付額外費用（如 5-10%）。

**避免清算**

- 不要借到極限
- 監控抵押率
- 準備追加抵押品或還款

### 利率機制

DeFi 借貸的利率是動態的：

**利用率**

池中有多少比例被借出：
- 總存款 $1,000,000
- 已借出 $700,000
- 利用率 = 70%

**利率曲線**

- 利用率低 → 利率低（鼓勵借款）
- 利用率高 → 利率飆升（鼓勵還款、吸引存款）

**目的**

確保池中總有流動性讓存款者提款。

- 功能豐富

**其他協議**

- Benqi（Avalanche）

### 存款收益

**收益來源**

- 借款者支付的利息

**收益率（APY）**

取決於：
- 資產類型
- 市場供需
- 協議激勵

**穩定幣 vs 波動資產**

- 穩定幣：通常 2-10% APY
- ETH：通常 0-5% APY
- 波動代幣：變化大

### 借款用途

為什麼要借款而不是直接賣出資產？

**1. 不想賣出**

看好 ETH，但需要資金：
- 抵押 ETH 借穩定幣
- 保留 ETH 的上漲潛力

**2. 稅務優化**

在某些司法管轄區，借款不是應稅事件（賣出是）。

**3. 槓桿**

借出資金買入更多資產（風險高）。

**4. 流動性需求**

臨時需要資金，不想變現長期持有的資產。

### 閃電貸

DeFi 獨有的創新：**閃電貸（Flash Loan）**。

**什麼是閃電貸？**

- 借款和還款在同一交易內完成
- 不需要抵押品
- 如果無法還款，整個交易回滾

**用途**
- 套利
- 清算
- 抵押品交換

**例子**

1. 閃電貸借 $1,000,000
2. 在 A 交易所以 $2,000 買入 ETH
3. 在 B 交易所以 $2,010 賣出 ETH
4. 還款 + 手續費
5. 保留利潤

如果第 3 步失敗，整個交易取消，就像沒發生過。

### 風險

**1. 清算風險**

價格波動可能導致清算。

**2. 智能合約風險**

協議可能有漏洞。

**3. 預言機風險**

價格資料來源出問題可能導致異常清算。

**4. 利率風險**

借款利率可能突然飆升。

**5. 流動性風險**

極端情況下可能無法提款。

### 如何開始？

**存款**

1. 訪問借貸協議（如 Aave）
2. 連接錢包
3. 選擇要存入的資產
4. 存入並開始賺取利息

**借款**

1. 先存入抵押品
2. 選擇要借的資產
3. 確認借款金額（注意抵押率）
4. 完成借款

### 進階策略

**遞歸借貸（Loop）**

1. 存入 $1,000 ETH
2. 借出 $600 穩定幣
3. 用穩定幣買 ETH
4. 再存入，再借出...
5. 放大曝險

風險很高，新手不建議。

**流動性挖礦**

借貸協議常有額外的代幣獎勵：
- 存款賺利息 + 代幣
- 借款付利息 + 獲得代幣
- 有時借款的代幣獎勵 > 利息支出

### 與傳統借貸比較

| 面向 | DeFi 借貸 | 傳統借貸 |
|------|-----------|----------|
| 審批 | 無 | 需要 |
| 速度 | 即時 | 數天 |
| 24/7 | 是 | 否 |
| 抵押 | 加密資產 | 房產/收入 |
| 匿名 | 是 | 否 |
| 利率 | 動態 | 固定/浮動 |
| 追討 | 無（自動清算）| 法律途徑 |
