# Web3

### Web1 是唯讀的（你只能看）。Web2 是可讀可寫的（你可以發文、按讚）。Web3 是可讀可寫可擁有的（你真正擁有你的數據和資產）。這就是 Web3 的核心理念。

---

## Web 演進史

| 時代 | 特點 | 代表 |
|------|------|------|
| Web1 | 唯讀 | 靜態網頁、Yahoo |
| Web2 | 讀寫 | Facebook、YouTube |
| Web3 | 讀寫擁有 | DeFi、NFT、DAO |

### 詳細比較

```
網際網路演進：

Web1（1990s-2000s）：
- 靜態網頁
- 只能閱讀
- 內容由少數人創建

Web2（2000s-now）：
- 社交媒體
- 用戶生成內容
- 平台擁有數據和流量

Web3（now-future）：
- 區塊鏈基礎
- 用戶擁有資產
- 去中心化治理
```

{% hint style="info" %}
**「擁有」的意義**

在 Web2，你在 Instagram 發的照片、在 Spotify 買的音樂，其實都不是你的——平台可以刪除、可以封號。在 Web3，你的 NFT、你的代幣，真正屬於你。
{% endhint %}

## Web3 核心概念

### 關鍵元素

| 概念 | 說明 |
|------|------|
| 區塊鏈 | 底層基礎設施 |
| 錢包 | 身份和資產管理 |
| 智能合約 | 自動執行的規則 |
| 代幣 | 價值和權利的載體 |
| 去中心化 | 沒有中央控制 |

### 技術堆疊

```
Web3 技術堆疊：

用戶層：
├── 錢包（MetaMask、Phantom）
└── DApp 前端

協議層：
├── DeFi 協議
├── NFT 標準
└── 身份系統

基礎設施層：
├── Layer 1（Ethereum、Solana）
├── Layer 2（Arbitrum、Optimism）
└── 儲存（IPFS、Arweave）
```

## Web3 的挑戰

### 目前問題

| 挑戰 | 說明 |
|------|------|
| 使用門檻 | 錢包、私鑰太複雜 |
| 擴展性 | 速度慢、費用高 |
| 監管 | 法規不明確 |
| 安全 | 駭客攻擊頻繁 |
| 用戶體驗 | 比 Web2 差很多 |

### 批評聲音

```
對 Web3 的批評：

「只是炒作」：
- 很多項目是龐氏騙局
- 實際應用有限
- 用戶大多是投機者

「不夠去中心化」：
- Infura、Alchemy 壟斷節點服務
- 大多數 NFT 圖片存在中心化伺服器
- VC 持有大量代幣

「技術退步」：
- 交易速度不如 Web2
- 用戶體驗差
- 解決的問題不存在
```

{% hint style="warning" %}
**保持懷疑**

Web3 有很多炒作和騙局。不是所有標榜 Web3 的項目都是好項目。要分辨真正的創新和包裝過的投機。
{% endhint %}
