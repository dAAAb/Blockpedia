# mBridge
> **最後更新：2026 年 1 月**


## mBridge 是由多國央行共同開發的跨境 CBDC 平台，目標是讓不同國家的央行數位貨幣可以互相兌換、即時結算。

在「[CBDC 總覽](README.md)」中，我們討論了各國自己的 CBDC。但更大的問題是：**不同國家的 CBDC 如何對話？**

這就是 mBridge 要解決的問題。

### 跨境支付的痛點

現在的跨境支付有多痛苦？

假設你要從台灣匯 1000 美元給美國的朋友：

```
你的台幣帳戶
→ 台灣的銀行
→ 代理銀行 A
→ SWIFT 系統
→ 代理銀行 B
→ 美國的銀行
→ 朋友的美元帳戶
```

這個過程：
- **耗時 3-5 個工作天**
- **手續費 25-50 美元**
- **匯率不透明**
- **中間可能出錯**

為什麼這麼複雜？因為不同國家的銀行系統不互通，需要靠「代理銀行」層層轉帳。

### mBridge 的願景

mBridge 想要這樣：

```
你的數位人民幣錢包
→ mBridge 平台
→ 對方的數位港幣錢包
```

- **秒級完成**
- **手續費極低**
- **匯率透明**
- **全程可追蹤**

簡單說，就像發微信一樣簡單，但傳的是錢。

### 參與國家

mBridge 由國際清算銀行（BIS）創新中心協調，參與者包括：

| 國家/地區 | 央行 | 貨幣 |
|----------|------|------|
| 中國 | 人民銀行 | 數位人民幣 |
| 香港 | 金融管理局 | 數位港幣 |
| 泰國 | 泰國央行 | 數位泰銖 |
| 阿聯酋 | 央行 | 數位迪拉姆 |
| 沙烏地阿拉伯 | 央行 | 數位里亞爾 |

2024 年，沙烏地阿拉伯正式加入，成為第五個成員。

### 技術架構

mBridge 使用自己開發的區塊鏈平台：**mBridge Ledger**。

**核心特點**：

**1. 多幣種支援**

平台上可以同時存在多種 CBDC，互相兌換。

**2. 原子結算**

交易要麼全部成功，要麼全部失敗，不會有中間狀態。這確保了安全性。

**3. 央行節點**

每個參與央行都運行自己的節點，共同維護帳本。沒有單一控制者。

**4. 隱私保護**

交易細節只有相關方可見，其他央行看不到。

**5. 合規設計**

內建 AML（反洗錢）檢查，符合國際金融監管要求。

### 試點成果

mBridge 已完成多輪試點：

**2022 年試點**

- 20 家商業銀行參與
- 完成超過 1.6 億美元真實交易
- 結算時間從數天縮短到秒級

**2024 年「最小可行產品」（MVP）**

- 擴大參與銀行範圍
- 測試更多交易場景
- 沙烏地阿拉伯加入

**未來規劃**

- 更多國家加入
- 更多貨幣對
- 可能與私人機構對接

{% hint style="info" %}
mBridge 是目前最成熟的跨境 CBDC 計畫，有可能成為未來國際支付的基礎設施。
{% endhint %}

### 應用場景

**1. 貿易結算**

中國企業向阿聯酋購買石油。

傳統方式：用美元結算，經過多個代理銀行。

mBridge 方式：直接用數位人民幣兌換數位迪拉姆，秒級完成。

**2. 僑匯**

泰國有大量海外勞工，每年匯回大量外匯。

用 mBridge，匯款成本可以大幅降低，勞工家庭受益。

**3. 企業資金調度**

跨國企業在多國有資金，需要頻繁調度。

mBridge 讓資金調度更快、更便宜。

**4. 央行外匯操作**

央行之間的外匯交易也可以用 mBridge，更有效率。

### 地緣政治意涵

mBridge 不只是技術項目，背後有深刻的地緣政治意涵。

**挑戰美元霸權？**

現在國際貿易大多用美元結算，美國因此擁有巨大優勢——包括制裁他國的能力。

mBridge 讓國家可以繞過美元直接交易。如果石油交易可以用人民幣-迪拉姆直接結算，美元的地位會動搖嗎？

這是美國密切關注 mBridge 的原因。

**新的金融秩序？**

mBridge 的參與者主要是亞洲和中東國家。如果這個系統成功，可能形成「非美元區」的金融體系。

當然，這還是很遠的事。美元的地位不會輕易動搖。但 mBridge 代表了一種可能性。

### 與 SWIFT 的關係

SWIFT 是現在國際支付的主要通道。mBridge 會取代它嗎？

短期內不會。

- SWIFT 連接全球 11,000+ 家金融機構
- SWIFT 有數十年的運營經驗
- 大部分國家還沒有 CBDC

但長期來看，如果越來越多國家有 CBDC，mBridge 這類平台可能蠶食 SWIFT 的市場。

SWIFT 自己也在研究如何整合 CBDC，保持相關性。

### 其他跨境 CBDC 計畫

mBridge 不是唯一的跨境 CBDC 計畫：

**Project Dunbar**
- 參與者：新加坡、澳洲、南非、馬來西亞
- 重點：多幣種結算平台

**Project Jura**
- 參與者：法國、瑞士
- 重點：歐元-瑞士法郎跨境結算

**Project Icebreaker**
- 參與者：以色列、挪威、瑞典
- 重點：零售 CBDC 跨境支付

**Project Mariana**
- 參與者：法國、新加坡、瑞士
- 重點：用 DeFi 技術進行跨境結算

這些計畫各有特色，未來可能互相連接，也可能形成競爭。

### 挑戰

**1. 治理問題**

多國共同運營，誰說了算？規則怎麼制定？爭議怎麼解決？

**2. 互操作性**

不同國家的 CBDC 設計不同，要怎麼兼容？

**3. 監管協調**

各國法規不同，跨境交易要遵守哪國的規定？

**4. 政治風險**

參與國之間如果發生政治衝突，系統還能繼續運作嗎？

**5. 擴展性**

目前只有 5 個參與者。如果擴展到 50、100 個，系統能承受嗎？

### 對我們的影響

如果你是：

**跨國企業**：未來可能享受更便宜、更快的跨境支付。

**外匯交易者**：市場結構可能改變，需要適應新規則。

**一般人**：出國旅遊、海外購物可能變得更方便。

**加密貨幣愛好者**：CBDC 的跨境整合可能對穩定幣市場產生影響。

### 未來展望

mBridge 的路線圖：

1. **擴大參與**：更多國家央行加入
2. **增加貨幣對**：支援更多 CBDC
3. **商業化運營**：從試點走向正式運營
4. **生態系統**：可能讓私人機構接入

這是一個漸進的過程，可能需要數年甚至十年。

但毫無疑問，跨境支付的未來正在被改寫。
