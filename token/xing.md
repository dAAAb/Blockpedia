# 星雲鏈（Nebulas）

## Nebulas（星雲鏈）是一個試圖為區塊鏈建立「搜尋引擎」的項目——它想為區塊鏈上的數據和 DApp 建立排名系統，就像 Google 為網頁排名一樣。它是 2017-2018 年中國區塊鏈項目的代表之一。

Google 如何決定搜尋結果的排名？

**PageRank 演算法。**

Nebulas 的創辦人問：區塊鏈世界能不能有類似的東西？

- 哪些智能合約更重要？
- 哪些 DApp 更有價值？
- 哪些地址更有影響力？

這就是 Nebulas 想解決的問題。

### 核心概念

#### Nebulas Rank（星雲指數）

Nebulas 的核心是一個排名演算法，用來衡量區塊鏈上地址的「價值」。

**考量因素**：

| 因素 | 說明 |
|------|------|
| 流動性 | 資產的交易活躍度 |
| 傳播性 | 影響力的傳遞範圍 |
| 互操作性 | 與其他地址的互動 |

**用白話說**：越多人用、越活躍、影響越大的地址，排名越高。

#### 為什麼這很重要？

**理論上的用途**：

- DApp 發現：找到最好用的應用
- 空投分配：獎勵真正的用戶
- 信用評分：區塊鏈上的「信用分數」
- 搜尋排名：區塊鏈版的 Google

### 創辦人背景

徐義吉是小蟻（NEO，原名 Antshares）的聯合創辦人。

| 時間 | 事件 |
|------|------|
| 2014 | 參與創立小蟻（NEO）|
| 2017 | 離開 NEO，創立 Nebulas |
| 2018 | Nebulas 主網上線 |

### 2017-2018 年的熱潮

Nebulas 是 ICO 熱潮中的明星項目：

| 時間 | 價格 |
|------|------|
| 2017 | ICO 價格約 $2 |
| 2018.1 | 歷史高點約 $15 |
| 2018.12 | 跌至 $1 以下 |

{% hint style="warning" %}

{% endhint %}

### 技術特色

#### 1. 可升級的協議

Nebulas 設計了一種機制，讓區塊鏈可以自我升級而不需要硬分叉。

#### 2. 開發者激勵計劃

「Incentive Program」獎勵開發者在 Nebulas 上建立 DApp。

#### 3. 多語言支援

智能合約支援 JavaScript 和 TypeScript，降低開發門檻。

### 為什麼沒有成功？

#### 1. 需求不明確

「區塊鏈搜尋引擎」聽起來很酷，但：

- 區塊鏈上的 DApp 數量本來就不多
- 用戶真的需要這個功能嗎？
- 市場需求未被驗證

#### 2. 執行困難

排名演算法的挑戰：

- 如何防止操縱？
- 如何定義「價值」？
- 數據來源如何確保？

#### 3. 生態系統弱

沒有足夠的 DApp 和用戶，排名系統就沒有意義。

#### 4. 熊市衝擊

2018 年熊市摧毀了大部分 ICO 項目。

### 現況

### 投資教訓

Nebulas 是一個很好的案例研究：

#### 1. 願景 ≠ 執行

好的想法不代表能實現。「區塊鏈 Google」聽起來很棒，但執行起來困難重重。

#### 2. 時機很重要

即使概念正確，如果市場還沒準備好，也很難成功。

#### 3. 熊市是殺手

90%+ 的 2017-2018 年 ICO 項目在熊市中消亡。

#### 4. 創辦人背景不是保證

徐義吉有 NEO 的經歷，但這不能保證新項目成功。

{% hint style="danger" %}
NAS 從 $15 跌到 $0.05，跌幅超過 99%。投資 ICO 項目就像投資早期創業公司——大部分會失敗。只用你能承受損失的錢投資，而且要有心理準備歸零。
{% endhint %}

### Nebulas 的歷史意義

雖然商業上不成功，Nebulas 提出的問題仍有價值：

- 區塊鏈上如何衡量「價值」？
- 去中心化的排名系統是否可能？
- 區塊鏈需要怎樣的「搜尋」功能？

這些問題在未來可能會有新的解答。
