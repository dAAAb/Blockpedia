# 比特黃金

### 比特幣挖礦被專業礦機壟斷，普通人根本沒機會參與。比特黃金想改變這件事——讓你用家裡的顯示卡，也能挖到比特幣的「黃金版」。但這個願景，真的實現了嗎？

---

## 什麼是比特黃金？

| 項目 | 說明 |
|------|------|
| 英文名 | Bitcoin Gold |
| 代號 | BTG |
| 分叉時間 | 2017 年 10 月 24 日 |
| 分叉高度 | 區塊 491,407 |
| 核心主張 | 抗 ASIC，讓挖礦更民主 |

## 為什麼要分叉？

### 比特幣的挖礦問題

```
分叉的背景：

2017 年的比特幣挖礦：

問題：
1. ASIC 礦機壟斷
   - 專業礦機效率是 GPU 的 100 倍+
   - 普通人買不起、買不到

2. 算力中心化
   - 幾家大礦場控制大部分算力
   - 主要集中在中國

3. 違背初衷
   - 中本聰設想「一 CPU 一票」
   - 現實變成「一 ASIC 一票」

比特黃金的解決方案：
- 換用抗 ASIC 的算法
- 讓 GPU 也能有效挖礦
- 「還礦於民」
```

### 技術差異

| 項目 | 比特幣 | 比特黃金 |
|------|--------|----------|
| 挖礦算法 | SHA-256 | Equihash → Zhash |
| 挖礦硬體 | ASIC | GPU |
| 區塊時間 | 10 分鐘 | 10 分鐘 |
| 總量 | 2,100 萬 | 2,100 萬 |

{% hint style="info" %}
**抗 ASIC 算法**

Equihash 是一種「記憶體困難」的算法，需要大量記憶體才能有效運算。這讓製造專用 ASIC 的成本大幅提高，GPU 反而更有優勢。
{% endhint %}

## 發展歷程

### 重要事件

| 時間 | 事件 |
|------|------|
| 2017/10 | 從比特幣分叉 |
| 2017/11 | 遭遇網站攻擊 |
| 2018/5 | 遭受 51% 攻擊，損失 1,800 萬美元 |
| 2019/1 | 第二次 51% 攻擊 |
| 2020 | 更換為 Zhash 算法 |

### 51% 攻擊事件

```
2018 年 5 月 51% 攻擊：

發生了什麼？
1. 攻擊者租用大量算力
2. 取得超過 50% 的網路算力
3. 進行雙重支付攻擊
4. 在交易所存入 BTG
5. 提領其他幣種
6. 回滾自己的存款交易

損失：
- 約 388,000 BTG
- 當時價值約 1,800 萬美元

教訓：
- 算力較低的幣種容易被攻擊
- 「民主化挖礦」也有代價
- 安全性需要足夠的算力保護
```

{% hint style="danger" %}
**安全性問題**

比特黃金因為算力相對較低，多次遭受 51% 攻擊。這是「抗 ASIC」策略的副作用——沒有專用礦機，也意味著更容易租用算力攻擊。
{% endhint %}

## 分叉幣的困境

### 共同問題

| 問題 | 說明 |
|------|------|
| 缺乏認可 | 不被視為「真正的比特幣」|

| Bitcoin Cash | 仍有一定市值 |
| Bitcoin Gold | 市值大幅下跌 |
| Bitcoin SV | 爭議極大 |
| 其他分叉 | 大多已無人問津 |

## 投資考量

### 優勢與風險

| 優勢 | 風險 |
|------|------|
| GPU 可挖 | 多次被 51% 攻擊 |
| 比特幣程式碼基礎 | 市值持續下滑 |
| 去中心化理念 | 生態系統薄弱 |
| 低價入手 | 可能歸零 |

{% hint style="danger" %}
**高風險警告**

比特黃金從 2017 年高點已下跌超過 99%。多次安全事件、缺乏採用、持續下跌的市值，都是嚴重的風險訊號。
{% endhint %}
