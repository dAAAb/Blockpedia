# 零幣

### 比特幣的交易紀錄是公開的，任何人都能追蹤你的錢包。零幣不一樣——它用數學魔法讓你證明「我有錢」，卻不需要告訴任何人你有多少錢、給了誰。

## 隱私技術

### 零知識證明

```
什麼是零知識證明？

比喻：色盲測試

情境：
你想證明你不是色盲，但不想透露你看到什麼顏色

傳統方式：
「我看到紅色」→ 洩露了資訊

零知識證明：
你能持續正確區分兩種顏色
→ 證明你不是色盲
→ 但對方不知道你看到什麼

在零幣中：
- 你能證明「我有足夠的錢付款」
- 但不透露你有多少錢、付給誰
```

### 兩種地址類型

| 類型 | 說明 |
|------|------|
| 透明地址（t-addr）| 類似比特幣，公開可查 |
| 隱蔽地址（z-addr）| 完全隱私，無法追蹤 |

{% hint style="info" %}
**選擇性隱私**

零幣讓用戶自己選擇——要公開透明還是完全隱私。這種設計讓它比其他隱私幣更靈活，也更容易符合監管要求。
{% endhint %}

## zk-SNARKs 原理

### 技術說明

```
zk-SNARKs 是什麼？

全名：
Zero-Knowledge Succinct Non-Interactive Argument of Knowledge
零知識簡潔非互動式知識論證

特性：
- Zero-Knowledge：不洩露額外資訊
- Succinct：證明很小，驗證很快
- Non-Interactive：不需要來回溝通

運作方式（簡化）：
1. 發送者生成加密證明
2. 證明包含：交易有效、餘額足夠
3. 網路驗證證明，不知道實際內容
4. 交易完成，無人知道金額和對象
```

### 與其他隱私幣比較

| 幣種 | 隱私技術 | 特色 |
|------|----------|------|
| Zcash | zk-SNARKs | 數學證明，最強隱私 |
| Monero | 環簽名 + 隱匿地址 | 強制隱私 |
| Dash | CoinJoin | 混幣，隱私較弱 |

## 發展歷程

### 重要事件

| 時間 | 事件 |

| 2016/10 | 主網上線，初期價格極高 |

### 爭議

| 議題 | 說明 |
|------|------|
| 信任設置 | 初始參數需要信任 |
| 開發者獎勵 | 20% 區塊獎勵給團隊 |
| 監管風險 | 部分國家禁止隱私幣 |
| 使用率 | 大多數交易仍用透明地址 |

{% hint style="warning" %}
**信任設置問題**

zk-SNARKs 需要初始「信任設置」儀式。如果參與者保留了某些資訊，理論上可以偽造證明。Zcash 使用多方計算降低風險。
{% endhint %}

## 監管與合規

### 隱私幣的困境

| 挑戰 | 說明 |
|------|------|
| 交易所下架 | 多國要求下架隱私幣 |
| 洗錢疑慮 | 被認為可能被濫用 |
| 合規壓力 | 很難符合 KYC/AML |
| 兩難 | 隱私權 vs 監管需求 |

{% hint style="danger" %}
**監管風險**

日本、韓國、澳洲等地的交易所已下架零幣。持有隱私幣可能面臨法律風險，請了解當地法規。
{% endhint %}
