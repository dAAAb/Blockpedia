# dAAAb（葛如鈞／寶博士）寫作風格指南

本文件記錄 dAAAb 在 Blockpedia 中的獨特寫作風格，供 Claude Code 學習與參考。

**重要原則**：dAAAb 的原創內容是 Gen AI 時代之前的人類思考結晶，未經允許不得改動任何一字。

---

## 一、文章結構模式

### 1. 開頭定義（必備）
每篇文章以 **粗體** 開頭，直接給出該術語的精準定義：

```markdown
# 區塊鏈

### 區塊鏈（Blockchain）就是經過密碼學加持的帳本創建及維護的工具。
```

```markdown
# 幣

## 幣（Money），即是一種共同約定的工作證明與交易記錄。
```

### 2. 章節結構
- H1（#）：術語名稱
- H3（###）：子主題，用於展開說明
- 使用 GitBook 的 `{% hint %}` 做提示框
- 文末附參考資料和嵌入連結

---

## 二、核心寫作手法

### 1. 比喻類比法（招牌特色）
用生活化、跨文化的比喻讓複雜概念變簡單：

| 概念 | 比喻 |
|------|------|
| 貨幣起源 | 「雞幣」故事 — 用兩條魚換一隻雞 |
| 區塊鏈 | 哈利波特的「魔法帳本」 |
| 分散式帳本 | 佛地魔的「分靈體」 |
| 中心化 vs 去中心化 | 漩渦鳴人的「影分身術」（戳中本體全滅 = 中心化） |
| 中心化帳本問題 | 「村長問題」— 村長半夜改帳本 |
| 比特幣 | 「數位版的黃金」 |

### 2. 敘事說故事
從歷史或虛構情境切入，建立讀者的情感連結：

> 想像一種情況，你肚子餓了，想吃雞，低頭看看...手裡剛去湖邊打來的幾條魚，你已經吃魚吃兩禮拜，實在想換個口味了，聽說隔壁村寶博士養的雞又肥又好吃。

### 3. 對話式口吻
直接與讀者對話，製造親切感：

- 「看見了嗎？」
- 「你可能在上一個章節提到佛地魔時就想問了吧？」
- 「這是個好問題，當然要誘之以利。」
- 「怎麼辦呢？」

### 4. 自稱「寶博士」
在文中以第三人稱提及自己：

> 寶博士常跟朋友說，如果這世界可能發生危難、災變或第三次世界大戰，發生前一定要準備一些比特幣。

## 三、文章元素

### 1. 表格比較
用表格呈現對比，清晰易懂：

```markdown
|       | **比特幣** | **黃金** |
|-------|-----------|---------|
| 總量限制 | ✔️ 有     | ✔️ 有   |
```

### 2. GitBook 提示框
- `{% hint style="info" %}` — 補充說明
- `{% hint style="success" %}` — 小八卦、趣聞
- `{% hint style="danger" %}` — 警告、風險提醒

### 3. 互動式學習
使用 `{% tabs %}` 製作「小挑戰」與隱藏答案：

```markdown
{% tabs %}
{% tab title="0x1 小挑戰" %}
現實社會中，是不是依舊存在類似雅浦島的中心化管理問題呢？誰是當前世界各國的村長？
{% endtab %}
{% tab title="答案" %}
政府和銀行。
{% endtab %}
{% endtabs %}
```

### 4. 嵌入式參考資料
文末嵌入 TED Talks、官方文件、外部文章：

```markdown
{% embed url="https://www.ted.com/talks/..." %}
其中有提到 Yap 石幣的故事，值得一看
{% endembed %}
```

### 5. 內部連結
大量使用 wiki 內部連結建立知識網絡：

```markdown
續「[帳本](ledger.md)」條目所言...
詳見「[挖礦](wa/)」條目...
```

## 四、語言風格

### 1. 中英對照
術語首次出現時附上英文，但標題純中文：

> 去中心化（英語：Decentralize）是中心化的相反...

### 2. 口語化但不失專業
- 使用「啊」「呀」「囉」「吧」等語氣詞
- 偶爾幽默：「村民們都累死了，整天轉帳都不用打魚養雞了」
- 但技術細節精準無誤

### 3. 正體中文用語
- 使用台灣正體中文
- 「帳本」而非「账本」
- 「區塊鏈」而非「区块链」

### 4. 反問與設問
頻繁使用修辭問句引導思考：

> 比特幣有價值嗎？
> 看到這裡，我相信你還是會很想問這個問題。就讓你問吧！

## 五、知識架構設計理念

### 1. 循序漸進的知識切入
從最基礎的「錢」「帳本」開始，而非直接講區塊鏈：

> 條目順序經過寶博士精心設計...讓人閱讀第 21 條時所需要的知識，必在第 21 條前已講述過。

### 2. 重視背景脈絡
不只講「是什麼」，更講「為什麼」：

- 為什麼需要貨幣？→ 延遲交換
- 為什麼需要帳本？→ 交易頻繁難追蹤
- 為什麼需要去中心化？→ 村長會作弊

### 3. 文化迷因融入
讓讀者不只懂技術，還能「入圈」：

> 希望讓一些圈子無論線下活動或線上社群裡口語表達的內容，和談話的材料，也能輕易被掌握、發揮並理解。
