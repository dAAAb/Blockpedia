# DIY 冷錢包（FireFly, HITCON etc.）

### 不想買現成的？自己做一個！DIY 冷錢包讓你從零開始理解硬體錢包的原理。這不只是省錢，更是一場深度學習之旅。台灣的 HITCON 和 FireFly 社群都有相關專案。

---

## 為什麼要 DIY

### DIY 的意義

```
自己做冷錢包的理由：

1. 學習
   - 深入理解原理
   - 密碼學實作
   - 硬體開發經驗

2. 安全考量
   - 不信任商業產品
   - 擔心後門
   - 完全掌控

3. 社群參與
   - 開源專案貢獻
   - 黑客松作品
   - 技術交流

4. 成就感
   - 自己做的最安心
   - 獨特的作品
```

{% hint style="warning" %}
**重要提醒**

DIY 冷錢包適合學習和實驗，但如果你要保護大量資產，建議還是使用經過市場驗證的商業產品。自製設備可能有未發現的安全漏洞。
{% endhint %}

## Arduino 方案

### 基本架構

```
Arduino 冷錢包組成：

硬體：
- Arduino 開發板（Uno, Nano 等）
- OLED 顯示螢幕
- 按鈕（確認/取消）
- （可選）SD 卡模組

軟體：
- 橢圓曲線加密庫
- BIP39 種子詞庫
- 交易簽署邏輯

功能：
- 產生種子詞
- 衍生私鑰/公鑰
- 簽署交易
- 顯示地址
```

```

- 運算能力強
```

## 開發流程

## 安全考量

### 常見漏洞

```
DIY 冷錢包的安全風險：

軟體層面：
- 隨機數生成不良
- 記憶體洩漏
- 側通道攻擊
- 時序攻擊

硬體層面：
- 無安全晶片
- 可能被物理提取
- 電磁洩漏

供應鏈：
- 元件可能被動手腳
- 需要可信賴的來源
```

### 防護建議

| 風險 | 防護 |
|------|------|
| 隨機數 | 使用硬體 RNG + 多重熵源 |
| 側通道 | 恆定時間演算法 |
| 記憶體 | 使用後清零 |
| 物理 | 安全外殼、防拆設計 |

{% hint style="danger" %}
**安全警告**

自製冷錢包很難達到商業產品的安全水準。如果你的密碼學或安全知識不夠深入，自製設備可能存在嚴重漏洞。建議先用小額測試。
{% endhint %}
