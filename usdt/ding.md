# 錨定

### 穩定幣的「穩定」靠的是錨定——讓 1 USDT 等於 1 美元、1 DAI 等於 1 美元。但錨定不是魔法，它需要機制來維持。當機制失效，穩定幣就會「脫鉤」。

---

## 什麼是錨定？

| 概念 | 說明 |
|------|------|
| 英文 | Peg / Pegging |
| 定義 | 維持與目標價格相等 |
| 目標 | 通常是 1 美元 |
| 重要性 | 穩定幣的核心機制 |

---

## 錨定機制類型

### 不同方式

```
穩定幣如何維持錨定：

1. 法幣儲備（USDT、USDC）
   - 每發行 1 幣，銀行存 1 美元
   - 可以 1:1 兌換回美元
   - 信任：相信發行方有足夠儲備

2. 超額抵押（DAI）
   - 存入價值 > 借出價值
   - 例：存 150 美元 ETH，借 100 DAI
   - 信任：相信智能合約和清算機制

3. 算法穩定（曾經的 UST）
   - 透過算法調節供需
   - 鑄造/銷毀機制
   - 信任：相信算法能維持平衡
```

### 機制比較

| 類型 | 代表 | 優點 | 缺點 |
|------|------|------|------|
| 法幣儲備 | USDT、USDC | 簡單可靠 | 中心化 |
| 超額抵押 | DAI | 去中心化 | 資本效率低 |
| 算法 | UST（已崩）| 資本效率高 | 風險極高 |

{% hint style="info" %}
**沒有完美的錨定機制**

每種錨定機制都有取捨。法幣儲備最穩但最中心化，算法最去中心化但最不穩。選擇穩定幣時要理解這些權衡。
{% endhint %}

## 錨定如何運作

### 套利機制

```
錨定靠套利維持：

當 USDT 價格 > $1：
1. 套利者在 Tether 以 $1 買入 USDT
2. 在市場以 >$1 賣出
3. 賺取差價
4. 增加市場供應 → 價格回落

當 USDT 價格 < $1：
1. 套利者在市場以 <$1 買入 USDT
2. 在 Tether 以 $1 贖回美元
3. 賺取差價
4. 減少市場供應 → 價格回升

關鍵：
- 套利者的行為維持價格穩定
- 前提是發行方能夠兌換
```

### DAI 的運作

| 機制 | 說明 |
|------|------|
| 超額抵押 | 抵押品價值 > 借出 DAI |
| 穩定費 | 借 DAI 要付利息 |
| 清算 | 抵押率不足會被清算 |
| 套利 | 價格偏離時有套利空間 |

## 脫鉤事件

### 什麼是脫鉤

| 概念 | 說明 |
|------|------|
| 英文 | De-peg |
| 定義 | 穩定幣價格偏離目標 |
| 輕微 | 偏離 1-2% |
| 嚴重 | 偏離超過 5% |
| 崩盤 | 完全失去錨定 |

### 歷史案例

```
重大脫鉤事件：

UST 崩盤（2022 年 5 月）：
- 從 $1 跌到 $0.1 以下
- 最終歸零
- 損失超過 400 億美元

USDC 脫鉤（2023 年 3 月）：
- 矽谷銀行倒閉
- USDC 有 33 億存在該銀行
- 價格一度跌到 $0.87
- 後來恢復錨定

DAI 脫鉤：
- 多次輕微脫鉤
- 通常在極端市場波動時
- 大多能自動恢復
```

{% hint style="danger" %}
**脫鉤風險**

即使是「穩定」幣，也有脫鉤風險。UST 的崩盤證明算法穩定幣可能完全歸零。USDC 的事件則顯示法幣儲備也有銀行風險。
{% endhint %}

## 評估錨定強度

### 檢查要點

| 要點 | 說明 |
|------|------|
| 儲備透明度 | 能驗證儲備嗎？|
| 審計頻率 | 多久審計一次？|
| 贖回便利性 | 能輕易贖回嗎？|
| 歷史記錄 | 過去有脫鉤嗎？|
| 市值規模 | 大到不能倒？|

### 風險評級

| 類型 | 風險 |
|------|------|
| USDT | 中（透明度疑慮）|
| USDC | 低（監管合規）|
| DAI | 低到中（依賴抵押品）|
| 算法穩定幣 | 極高 |
