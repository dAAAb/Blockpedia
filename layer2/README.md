# Layer 2 擴容方案
> **最後更新：2026 年 1 月**


## Layer 2 是建立在以太坊之上的擴容網路，讓交易更快、更便宜，同時繼承以太坊的安全性。

以太坊的問題是什麼？

**太慢，太貴。**

一筆簡單的轉帳可能要 $10 手續費。
網路擁擠時，可能要等好幾分鐘。

這阻礙了大規模應用。

**Layer 2** 就是解決方案：在以太坊之上建立更快、更便宜的網路。

### 什麼是 Layer 2？

**Layer 1**：主區塊鏈（如以太坊）
**Layer 2**：建立在 Layer 1 之上的擴容網路

Layer 2 處理大量交易，然後把結果「提交」到 Layer 1。

**比喻**

Layer 1 像法院，處理案件慢但權威。
Layer 2 像調解中心，快速處理糾紛。
重要案件才送到法院（Layer 1）。

### 為什麼需要 Layer 2？

**區塊鏈三難困境**

區塊鏈很難同時達到：
1. **去中心化**
2. **安全性**
3. **可擴展性**

以太坊選擇了去中心化和安全性，犧牲了可擴展性。

**Layer 2 的解決方案**

- 在 Layer 2 上處理大量交易
- 把安全性「錨定」到 Layer 1
- 兼顧速度和安全

### Layer 2 的類型

**1. Optimistic Rollups**

「樂觀」地假設交易都是有效的。

- 交易先執行，有問題再處理
- 挑戰期（通常 7 天）
- 代表：[Optimism](optimism.md)、[Arbitrum](arbitrum.md)、[Base](base.md)

**2. ZK Rollups**

用零知識證明驗證交易。

- 每批交易都有數學證明
- 不需要挑戰期
- 代表：[zkSync](zksync.md)、[StarkNet](starknet.md)

**3. 其他**

- Validium：數據存在鏈下
- Plasma：較早的方案，現較少用
- State Channels：閃電網路類型

### Optimistic Rollups

**原理**

1. 用戶在 L2 發送交易
2. 排序器打包交易
3. 「樂觀地」假設交易有效
4. 把交易數據提交到 L1
5. 有挑戰期讓人檢舉問題

**挑戰期**

- 通常 7 天
- 任何人可以提交「欺詐證明」
- 如果發現問題，交易被回滾

**優點**
- 技術相對成熟
- 與以太坊高度兼容
- 已有大規模應用

**缺點**
- 提款需要等待挑戰期
- 依賴至少一個誠實的驗證者

### ZK Rollups

**原理**

1. 用戶在 L2 發送交易
2. 排序器執行交易
3. 生成零知識證明（ZKP）
4. 把證明和數據提交到 L1
5. L1 驗證證明

**零知識證明**

一種密碼學技術，可以證明某件事是真的，而不透露額外信息。

「我知道答案」→ 證明我真的知道 → 但不告訴你答案是什麼

**優點**
- 不需要挑戰期
- 理論上更安全
- 提款更快

**缺點**
- 技術更複雜
- 與 EVM 兼容性挑戰
- 還在發展中

### 比較表

| 特性 | Optimistic Rollups | ZK Rollups |
|------|-------------------|------------|
| 驗證方式 | 欺詐證明 | 有效性證明 |
| 提款時間 | 7 天 | 幾小時 |
| 計算成本 | 較低 | 較高 |
| EVM 兼容 | 高 | 發展中 |
| 成熟度 | 較高 | 發展中 |
| 代表 | Optimism, Arbitrum | zkSync, StarkNet |

### 主要 Layer 2

**Optimistic Rollups**

| L2 | 特色 | TVL 規模 |
|----|------|----------|
| [Arbitrum](arbitrum.md) | 生態最大 | 最大 |
| [Optimism](optimism.md) | OP Stack | 大 |

**ZK Rollups**

| L2 | 特色 | 狀態 |
|----|------|------|

| Polygon zkEVM | Polygon 生態 | 主網上線 |

### 使用 Layer 2

**1. 橋接資產**

把資產從 L1 轉到 L2：
- 使用官方橋
- 或使用第三方橋（更快但有風險）

**2. 在 L2 上操作**

- 設定錢包連接到 L2 網路
- 執行交易（更快、更便宜）
- 使用 L2 上的 DApp

**3. 提款到 L1**

- Optimistic Rollup：等待 7 天
- ZK Rollup：通常幾小時
- 或使用快速橋（支付額外費用）

### 橋接

**官方橋**

每個 L2 都有官方橋：
- 最安全
- 但可能較慢

**第三方橋**

- Hop Protocol
- Stargate
- Across
- Synapse

更快，但有額外風險。

**跨 L2 橋接**

從一個 L2 直接轉到另一個 L2：
- 不需要經過 L1
- 更快更便宜

### 費用比較

以簡單的 ETH 轉帳為例（參考值）：

| 網路 | 費用 |
|------|------|
| Ethereum L1 | $1-10 |
| Arbitrum | $0.01-0.10 |
| Optimism | $0.01-0.10 |
| Base | $0.001-0.01 |
| zkSync | $0.01-0.10 |

（費用會隨網路狀況變動）

### 安全性

**繼承 L1 安全性**

L2 的安全性來自 L1：
- 交易數據存在 L1
- 可以透過 L1 恢復資產
- 即使 L2 出問題，資產不會丟失

**但有前提**

- L2 需要正常運作
- 需要有人監控（Optimistic Rollup）
- 證明系統需要正確（ZK Rollup）

### 中心化問題

目前大多數 L2 有一定程度的中心化：

**排序器**

大多數 L2 有單一排序器：
- 負責打包交易
- 可能導致審查
- 正在走向去中心化

**升級權限**

L2 合約可以被升級：
- 安全理事會控制
- 需要信任這些實體
- 風險在減少中

### 未來發展

**1. 去中心化排序器**

讓排序器去中心化，減少審查風險。

**2. 更多 ZK 採用**

ZK 技術成熟後，可能成為主流。

**3. 互操作性**

L2 之間更順暢的資產和信息流動。

**4. Layer 3**

在 L2 之上建立更專門的網路。

**5. Proto-danksharding**

以太坊的 EIP-4844（blob）降低 L2 成本。

### 選擇哪個 L2？

**看生態**
- 你要用的 DApp 在哪個 L2？
- 流動性在哪裡？

**看費用**
- 不同 L2 費用不同
- 使用頻率高選更便宜的

**看安全性**
- 更成熟的 L2 相對更安全
- 了解去中心化程度

**看提款時間**
- 需要快速提款選 ZK Rollup
- 或使用快速橋

{% hint style="info" %}
Layer 2 是以太坊擴容的關鍵。它讓區塊鏈從「太慢太貴」變成真正可用。了解 Layer 2，是理解現代區塊鏈應用的必備知識。
{% endhint %}
