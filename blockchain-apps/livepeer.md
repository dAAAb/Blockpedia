# Livepeer

### YouTube、Netflix 每天處理數十億小時的影片，但背後的轉碼成本驚人。Livepeer 用去中心化的方式解決這個問題——讓全球閒置的 GPU 一起工作，降低影片處理成本高達 10 倍。

## 影片轉碼是什麼？

### 基本概念

```
為什麼需要影片轉碼：

問題：
- 用戶設備不同（手機、電視、電腦）
- 網路速度不同（4G、WiFi、光纖）
- 一個影片無法適用所有人

轉碼：
- 將原始影片轉換成多種格式
- 不同解析度（1080p、720p、480p）
- 不同編碼格式
- 適應不同設備和網速

傳統做法：
- 中心化資料中心
- 昂貴的伺服器
- 成本高昂
```

### Livepeer 的解決方案

| 傳統方式 | Livepeer |
|----------|----------|
| 中心化伺服器 | 分散式節點 |
| 高固定成本 | 按需付費 |
| 單一供應商 | 開放網路 |
| 容易壟斷 | 競爭市場 |

## 運作機制

### 網路角色

| 角色 | 功能 |
|------|------|
| Orchestrator | 協調轉碼工作 |
| Transcoder | 執行實際轉碼 |
| Broadcaster | 提交影片需求 |
| Delegator | 質押 LPT 支持節點 |

```
Livepeer 運作流程：

1. Broadcaster 提交影片
   - 上傳原始影片
   - 指定輸出格式
   - 支付 ETH/DAI

2. Orchestrator 接單
   - 競爭獲得工作
   - 分配給 Transcoder
   - 保證服務品質

3. Transcoder 執行
   - 利用 GPU 轉碼
   - 產出多種格式
   - 回傳結果

4. 結算獎勵
   - LPT 通膨獎勵
   - ETH 手續費分配
   - Delegator 獲得分潤
```
