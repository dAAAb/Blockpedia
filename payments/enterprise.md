# 企業加密支付

## 從 Tesla 接受比特幣到 Shopify 整合穩定幣，企業採用加密支付正從實驗走向實踐。這是 Web3 支付的下一個戰場。

2021 年，Tesla 宣布接受比特幣買車。

消息一出，比特幣價格飆升。

幾個月後，Tesla 又宣布暫停接受比特幣（理由是環保）。

價格又跌了。

這個故事說明了幾件事：
1. 企業對加密支付確實有興趣
2. 但還在摸索階段
3. 波動性是個問題

現在是 2025-2026 年，情況變得更成熟了。

### 企業為什麼考慮加密支付？

**1. 降低成本**

信用卡手續費 2-3%，穩定幣可能 < 1%。

對於利潤率低的行業（如電商），這是顯著的節省。

一個年營業額 1 億美元的公司，節省 2% 手續費就是 200 萬美元。

**2. 更快結算**

信用卡：T+1 到 T+2
穩定幣：T+0（即時）

更快拿到錢，改善現金流。

**3. 無拒付風險**

信用卡有「chargeback」問題——消費者可以申請退款，商家可能被扣錢。

區塊鏈交易不可逆。一旦付款，無法單方面撤銷。

這對某些高風險行業特別有價值。

**4. 跨境效率**

如果你有國際業務，穩定幣讓跨境收付款變得更簡單。

詳見「[跨境支付](cross-border.md)」。

**5. 吸引新客群**

加密貨幣持有者是一個獨特的消費族群。他們可能優先選擇接受加密貨幣的商家。

**6. 創新形象**

接受加密貨幣可以塑造「創新」「科技前瞻」的品牌形象。

### 企業採用的類型

企業可以在不同層面採用加密支付：

**收款（Accepting Payments）**

最基本的：接受客戶用加密貨幣付款。

方式：
- 整合 Coinbase Commerce、BitPay 等
- 使用 Stripe 的穩定幣功能（新推出）
- 自建解決方案

**付款（Making Payments）**

用加密貨幣付款給供應商、員工、合作夥伴。

場景：
- 支付海外供應商
- 給遠端員工發薪
- 支付行銷費用

**資金管理（Treasury）**

把公司資金的一部分用加密貨幣持有。

這跟「支付」不同，更像是投資/資產配置。

MicroStrategy 是最極端的例子——持有超過 20 萬枚比特幣。

### 實際案例

**Shopify**

- 整合多種加密支付選項
- 商戶可以選擇接受穩定幣
- 自動轉換功能

**微軟**

- Xbox 商店接受比特幣
- 透過 BitPay 處理

**AT&T**

- 2019 年開始接受比特幣帳單支付
- 透過 BitPay

**Twitch**

- 接受多種加密貨幣
- 主播可以收加密打賞

**Overstock**

- 最早接受比特幣的大型電商之一
- 2014 年開始

**PayPal 商戶**

- PayPal 的 Checkout with Crypto
- 讓消費者用加密貨幣付款（自動轉成法幣給商戶）

### 整合方案

企業想接受加密支付，有幾種選擇：

**Coinbase Commerce**

- 簡單整合
- 支援多種加密貨幣
- 可以自動轉換成法幣
- 手續費：1%

適合：中小企業、電商

**BitPay**

- 老牌加密支付服務
- 支援企業級需求
- 發票、退款等功能完整
- 有合規解決方案

適合：中大型企業

**Stripe（新功能）**

- 收購 Bridge 後推出穩定幣支付
- 對現有 Stripe 商戶最方便
- 穩定幣為主

適合：已經用 Stripe 的商戶

**Square / Block**

- Cash App 整合
- [Bitcoin for Business](block-btc.md)
- 比特幣為主

適合：實體店面

**自建方案**

大企業可能自建：
- 完全控制
- 客製化
- 但需要技術能力

適合：大型科技公司

### 會計和稅務

企業採用加密支付，會計和稅務是重要考量：

**收入確認**

收到加密貨幣時，如何確認收入？
- 通常按收款時的市價計算
- 需要記錄時間點和匯率

**資產分類**

持有的加密貨幣是什麼資產類別？
- 在很多地方，被視為「無形資產」
- 需要定期評估減值
- 稅務處理因地區而異

**增值稅 / 銷售稅**

用加密貨幣買東西，稅怎麼算？
- 通常按法幣價值計算
- 各地規定不同

**跨境稅務**

跨境加密支付的稅務更複雜。可能需要專業顧問。

{% hint style="info" %}
建議企業在採用加密支付前，諮詢專業會計師和稅務顧問。
{% endhint %}

### 合規要求

企業採用加密支付需要注意合規：

**KYC/AML**

反洗錢法規要求知道「誰在付款」。

使用託管服務（如 Coinbase Commerce）通常已經處理這個問題。

自建方案需要自己處理。

**支付牌照**

在某些地區，處理支付需要特定牌照。

使用第三方支付服務可以避免這個問題。

**資料保護**

區塊鏈上的交易是公開的。需要注意客戶隱私。

**行業特定規定**

某些行業（如金融、醫療）有額外的合規要求。

### 風險管理

**波動風險**

即使用穩定幣，也有小幅波動風險。

解決方案：即時轉換成法幣。

如果接受比特幣/以太幣，波動風險更大。

**對手方風險**

使用第三方服務，要考慮對方的信用風險。

選擇有信譽的服務商。

**技術風險**

智能合約漏洞、私鑰管理、系統故障等。

需要適當的技術和保險。

**監管風險**

政策可能改變。需要持續關注監管動態。

### 員工薪資

越來越多公司提供加密貨幣發薪選項：

**部分薪資**

員工可以選擇把部分薪資用加密貨幣領取。

比如：70% 法幣，30% 比特幣

**完全加密薪資**

某些加密公司完全用加密貨幣發薪。

**平台**：
- BitWage
- Deel（支援加密選項）
- Papaya Global

**員工考量**：
- 稅務處理（通常按收到時市價報稅）
- 波動風險
- 兌換便利性

### B2B 支付

企業對企業的支付，加密貨幣可能更有優勢：

**供應商付款**

特別是跨境供應商：
- 更快到帳
- 更低手續費
- 無需開立信用狀

**分潤 / 佣金**

平台支付給合作夥伴的分潤，用穩定幣可以：
- 即時結算
- 透明追蹤
- 自動化（智能合約）

**子公司間調度**

跨國企業在子公司間調動資金，傳統方式複雜。穩定幣可以簡化。

### 整合考量

企業整合加密支付時要考慮：

**1. 目標是什麼？**

- 降低成本？
- 吸引新客群？
- 跨境效率？
- 品牌形象？

不同目標，解決方案不同。

**2. 資源和能力**

- 有技術團隊嗎？
- 能處理合規嗎？
- 願意投入多少？

資源有限的公司，用現成方案（如 Coinbase Commerce）比較實際。

**3. 客戶需求**

你的客戶真的想用加密貨幣付款嗎？

可以先做調查，或者小規模試驗。

**4. 風險承受度**

能接受多大的波動風險？

風險厭惡的公司應該選擇即時轉換成法幣的方案。

**5. 監管環境**

你的行業和地區對加密支付的態度如何？

某些地區可能有限制。

### 成功因素

企業成功採用加密支付的關鍵：

**1. 從小開始**

不要一次全面導入。先在一個產品線、一個地區試驗。

**2. 提供選擇，不強迫**

加密支付應該是「額外選項」，不是「唯一方式」。

大部分客戶還是習慣傳統支付。

**3. 簡化流程**

對客戶來說，用加密付款應該跟用信用卡一樣簡單。

如果太複雜，沒人會用。

**4. 內部教育**

財務、法務、客服團隊都要了解加密支付的基本知識。

**5. 持續監控**

追蹤使用情況、成本節省、問題回報，持續優化。

### 未來趨勢

**1. 穩定幣主導**

企業支付會以穩定幣為主。波動的加密貨幣不適合日常商業。

**2. 自動化**

智能合約讓支付可以自動化：
- 達到條件自動付款
- 按比例自動分配
- 自動合規檢查

**3. 與 ERP 整合**

加密支付會整合進 SAP、Oracle 等企業系統。

**4. 即時結算成為標準**

T+2 會變成歷史。即時結算會成為預期。

**5. 更多大企業進入**

隨著監管明確、工具成熟，更多財富 500 強會採用加密支付。

### 行動建議

**如果你是小型企業**：
1. 評估是否有客戶需求
2. 考慮 Coinbase Commerce 或 BitPay
3. 從小規模試驗開始
4. 諮詢會計師了解稅務影響

**如果你是中型企業**：
1. 評估跨境支付需求（這通常是最大痛點）
2. 比較不同解決方案
3. 考慮合規需求
4. 建立內部流程和政策

**如果你是大型企業**：
1. 進行全面評估（技術、合規、稅務、營運）
2. 可能需要組建專門團隊
3. 考慮與現有支付基礎設施的整合
4. 可能需要自建部分解決方案

企業加密支付正從「創新嘗試」變成「實際選項」。關鍵是找到適合自己的方式，而不是盲目跟風。
