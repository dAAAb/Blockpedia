# 瞭解你的用戶（KYC）

### 想在交易所買幣？先證明你是誰。KYC 是「認識你的客戶」的意思，你需要提交身份證件讓平台驗證。這是監管要求，也是防止洗錢的手段。對於重視隱私的人來說，這是個兩難。

> ⚠️ **免責聲明**：本文僅供教育目的，不構成投資建議。加密貨幣投資有風險，請自行研究（DYOR）後再做決定。

---

## 什麼是 KYC？

| 概念 | 說明 |
|------|------|
| 全名 | Know Your Customer |
| 中文 | 認識你的客戶 / 實名認證 |
| 目的 | 驗證用戶身份 |
| 來源 | 金融監管要求 |

---

## 為什麼需要 KYC？

### 監管角度

```
KYC 的目的：

1. 防止洗錢（AML）
   - 確認資金來源
   - 追蹤可疑交易
   - 符合法規要求

2. 防止恐怖融資
   - 審查制裁名單
   - 阻止非法資金流動

3. 防止詐騙
   - 確認真實身份
   - 減少假帳戶
   - 保護其他用戶

4. 稅務追蹤
   - 政府可追蹤交易
   - 確保稅務合規
```

### 平台角度

| 原因 | 說明 |
|------|------|
| 法律要求 | 不 KYC 可能被罰 |
| 銀行合作 | 銀行要求合規 |
| 保險需求 | 降低風險 |
| 用戶保護 | 減少帳戶被盜風險 |

{% hint style="info" %}
**不是所有平台都要 KYC**

去中心化交易所（DEX）通常不需要 KYC，因為你直接用錢包交易。但中心化交易所（CEX）幾乎都要求 KYC。
{% endhint %}

## KYC 流程

### 常見步驟

```
KYC 驗證步驟：

基本驗證（Tier 1）：
- 姓名
- Email
- 手機號碼
- 提款額度較低

身份驗證（Tier 2）：
- 身份證/護照
- 地址證明
- 自拍照
- 較高提款額度

進階驗證（Tier 3）：
- 收入證明
- 資金來源
- 視訊驗證
- 最高額度或無限制
```

### 所需文件

| 文件類型 | 範例 |
|----------|------|
| 身份證明 | 護照、身份證、駕照 |
| 地址證明 | 水電帳單、銀行對帳單 |
| 自拍 | 手持證件自拍 |
| 其他 | 收入證明、資金來源 |

---

## KYC 等級與限制

### 常見分級

| 等級 | 驗證內容 | 提款限制 |
|------|----------|----------|
| 無 KYC | 無驗證 | 極低或禁止 |
| 基本 | Email + 手機 | 低 |
| 中級 | 身份證明 | 中 |
| 完整 | 全部驗證 | 高或無限 |

## 隱私 vs 合規

### 兩難困境

```
KYC 的爭議：

支持 KYC：
- 防止犯罪使用
- 提高市場信任
- 利於主流採用
- 保護用戶

反對 KYC：
- 侵犯隱私
- 資料外洩風險
- 違背去中心化精神
- 排除無證件者
```

### 資料外洩風險

| 事件 | 說明 |
|------|------|
| 交易所被駭 | KYC 資料可能外洩 |
| 內部濫用 | 員工可能存取資料 |
| 政府要求 | 可能被強制交出 |

{% hint style="warning" %}
**資料外洩風險**

多家交易所曾發生 KYC 資料外洩事件。你的身份證、護照照片可能落入不法份子手中，被用於身份盜竊或其他犯罪。
{% endhint %}

## 不同平台的 KYC 要求

### 比較

| 平台類型 | KYC 要求 |
|----------|----------|
| 大型 CEX（幣安、Coinbase）| 強制 |
| 中小型 CEX | 大多強制 |
| DEX（Uniswap）| 不需要 |
| P2P 平台 | 因平台而異 |
| 部分平台 | 低額度免 KYC |

## 如何保護 KYC 資料

### 自保建議

```
提交 KYC 時的注意事項：

選擇平台：
- 選擇信譽良好的交易所
- 查看過去有無資料外洩
- 了解其資料保護政策

提交時：
- 只提交必要資料
- 照片加浮水印（「僅供 XX 平台 KYC 使用」）
- 記錄提交了什麼

提交後：
- 定期檢查是否有異常
- 考慮信用監控服務
- 如有外洩，立即反應
```
