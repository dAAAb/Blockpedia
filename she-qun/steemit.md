# Steemit

### Steemit 是幣圈「內容挖礦」的先驅。在 2016 年，當大多數人還不知道區塊鏈是什麼的時候，Steemit 就在嘗試用代幣獎勵創作者和讀者。雖然它後來遭遇了不少挫折，但它開創的「創作即挖礦」模式影響了無數後來的 Web3 社群平台。

---

## Steemit 是什麼？

Steemit 是建立在 Steem 區塊鏈上的社群媒體平台，由 Ned Scott 和 Dan Larimer（後來創立 EOS 的人）於 2016 年創立。

```
Steemit 的核心概念：

傳統社群媒體：
你發文 → 平台賺廣告費 → 你得到讚

Steemit：
你發文 → 社群投票 → 你獲得 STEEM 代幣
```

| 代幣 | 功能 |
|------|------|
| STEEM | 基礎代幣，可交易 |
| Steem Power (SP) | 鎖倉的 STEEM，增加投票權重 |
| Steem Dollars (SBD) | 穩定幣，錨定 1 美元 |

## Steemit 的歷史

| 時間 | 事件 |
|------|------|
| 2016 年 | Steemit 上線，掀起熱潮 |
| 2017 年 | 牛市高峰，STEEM 價格飆漲 |
| 2018 年 | Dan Larimer 離開，專注 EOS |
| 2020 年 | Justin Sun（TRON 創辦人）收購 Steem |
| 2020 年 | 社群分叉出 Hive 區塊鏈 |

{% hint style="warning" %}
**Steem vs Hive 的分裂**

2020 年孫宇晨收購 Steem 後，引發社群強烈反彈。大量用戶和見證人（類似驗證者）分叉出 Hive 區塊鏈，形成「Steem 留守派」和「Hive 出走派」的對立。
{% endhint %}

## Steemit 的運作方式

### 獎勵機制

```
發文獎勵流程：

1. 作者發布文章
2. 讀者用 SP 投票（點讚）
3. 7 天後結算獎勵
4. 獎勵分配：
   - 作者獲得約 75%
   - 策展人（投票者）獲得約 25%
```

### 投票權重

| 因素 | 影響 |
|------|------|
| Steem Power | SP 越多，投票價值越高 |
| 投票時機 | 越早投票，策展獎勵越高 |
| 聲望值 | 高聲望者的投票更有影響力 |

## Steemit 的問題

### 為什麼它沒有成功？

| 問題 | 說明 |
|------|------|
| 大戶壟斷 | 少數鯨魚控制了大部分獎勵 |
| 機器人橫行 | 自動投票機器人削弱真實互動 |
| 內容農場 | 為了賺幣而大量產出低質量內容 |
| 治理問題 | 中心化控制引發社群分裂 |

```
理想 vs 現實：

理想：
好文章 → 社群投票 → 獲得獎勵 → 激勵更多好文章

現實：
互投互讚 → 鯨魚控制 → 機器人刷量 → 劣幣驅逐良幣
```

{% hint style="info" %}
**代幣經濟的教訓**

Steemit 證明了一件事：單純用代幣激勵無法保證內容品質。人們會優化「賺幣」而非「創作好內容」。這個教訓影響了後來所有的 Web3 社群設計。
{% endhint %}

## Steemit 的遺產

### 影響了什麼？

| 領域 | 影響 |
|------|------|
| SocialFi | 啟發了 Friend.tech 等社交金融應用 |
| 創作者經濟 | 證明了區塊鏈可以用於內容變現 |
| DPoS | 推廣了委託權益證明共識機制 |
| Hive | 分叉出更去中心化的社群 |

### 後繼者

| 平台 | 特色 |
|------|------|
| Hive | Steem 的分叉，更去中心化 |
| Mirror | 以太坊上的創作平台 |
| Lens Protocol | 去中心化社交圖譜 |
| Farcaster | 去中心化 Twitter 替代品 |

| 狀態 | 說明 |
