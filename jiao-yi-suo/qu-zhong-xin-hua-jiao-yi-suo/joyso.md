# JOYSO

### JOYSO 是 2018 年由台灣團隊推出的去中心化交易所，採用「鏈下撮合、鏈上結算」的混合架構，基於 0x 協議開發。它試圖解決純鏈上 DEX 速度慢、費用高的問題，是台灣在 DEX 領域的早期嘗試。

---

## JOYSO 的特色

### 混合式架構

```
傳統 CEX：全部中心化
    ↓ 效率高，但要信任交易所

純鏈上 DEX：全部去中心化
    ↓ 安全，但慢且貴

JOYSO：混合式
    ↓ 撮合快（鏈下），結算安全（鏈上）
```

| 操作 | 執行位置 | 特點 |
|------|----------|------|
| 掛單 | 鏈下 | 快速、免費 |
| 撮合 | 鏈下 | 即時配對 |
| 結算 | 鏈上 | 安全、透明 |
| 資產 | 鏈上 | 始終在用戶錢包 |

{% hint style="info" %}
**不用把幣存進去**

和中心化交易所不同，使用 JOYSO 時，你的資產始終在自己的以太坊錢包裡，交易時才透過智能合約轉移。
{% endhint %}

## JOYSO 的技術基礎

### 0x 協議

JOYSO 使用 0x 協議作為底層技術：

| 組件 | 說明 |
|------|------|
| 0x 智能合約 | 處理代幣交換 |
| 訂單格式 | 標準化的訂單結構 |
| 簽名驗證 | 確保訂單真實性 |

### 支援的代幣

主要支援以太坊上的 ERC-20 代幣交易對。

## JOYSO 的現況

{% hint style="warning" %}
**專案狀態**

JOYSO 在 2018-2019 年曾活躍運營，但隨著 DEX 競爭加劇和市場變化，目前已不是主流選擇。台灣用戶現在更常使用 Uniswap、SushiSwap 等國際化 DEX。
{% endhint %}

---

## 為什麼混合式 DEX 沒有成為主流？

| 因素 | 說明 |
|------|------|
| AMM 崛起 | Uniswap 的 AMM 模式更簡單 |
| 流動性問題 | 難以與大型 DEX 競爭 |
| 用戶習慣 | 大眾更熟悉 AMM 的交易體驗 |
| 生態系統 | AMM 更容易與 DeFi 整合 |

```
2018：訂單簿 DEX 是主流探索方向
2020：Uniswap 帶動 AMM 革命
2021+：AMM 成為 DEX 主流
```
